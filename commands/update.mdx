## Overview

The `conda update` command updates conda packages to the latest compatible version. It accepts a list of package names and updates them to the latest versions that are compatible with all other packages in the environment.

Conda attempts to install the newest versions of the requested packages. To accomplish this, it may update some packages that are already installed, or install additional packages.

## Syntax

```bash
conda update [options] [package_spec ...]
```

## Target Environment Specification

<ParamField path="-n, --name" type="string">
  Name of environment.
</ParamField>

<ParamField path="-p, --prefix" type="string">
  Full path to environment location (i.e. prefix).
</ParamField>

## Package Specifications

<ParamField path="package_spec" type="string">
  List of packages to update in the conda environment. Package specifications can include:
  - Simple names: `numpy`
  - Version constraints: `python>=3.11`, `numpy>=1.20`
</ParamField>

<ParamField path="--file" type="string">
  Read package versions from the given file. Repeated file specifications can be passed (e.g. `--file=file1 --file=file2`).
</ParamField>

## Solver Mode Modifiers

<ParamField path="--freeze-installed, --no-update-deps" type="boolean">
  Do not update or change already-installed dependencies. This may force conda to install older versions of the requested packages.
</ParamField>

<ParamField path="--update-deps" type="boolean">
  Update dependencies that have available updates.
</ParamField>

<ParamField path="-S, --satisfied-skip-solve" type="boolean">
  Exit early and do not run the solver if the requested specs are satisfied. Also skips aggressive updates as configured by the `aggressive_update_packages` config setting.
</ParamField>

<ParamField path="--update-all, --all" type="boolean">
  Update all installed packages in the environment.
</ParamField>

<ParamField path="--update-specs" type="boolean">
  Update based on provided specifications.
</ParamField>

<ParamField path="--no-deps" type="boolean">
  Do not install, update, remove, or change dependencies. This WILL lead to broken environments and inconsistent behavior. Use at your own risk.
</ParamField>

<ParamField path="--only-deps" type="boolean">
  Only update dependencies.
</ParamField>

<ParamField path="--force-reinstall" type="boolean">
  Ensure that any user-requested package for the current operation is uninstalled and reinstalled, even if that package already exists in the environment.
</ParamField>

<ParamField path="--no-pin" type="boolean">
  Ignore pinned file.
</ParamField>

<ParamField path="--strict-channel-priority" type="boolean">
  Packages in lower priority channels are not considered if a package with the same name appears in a higher priority channel.
</ParamField>

<ParamField path="--no-channel-priority" type="boolean">
  Package version takes precedence over channel priority. Overrides the value given by `conda config --show channel_priority`.
</ParamField>

<ParamField path="--solver" type="string">
  Choose which solver backend to use.
</ParamField>

## Channel Customization

<ParamField path="-c, --channel" type="string">
  Additional channel to search for packages. These are URLs searched in the order they are given (including local directories using the `file://` syntax or simply a path like `/home/conda/mychan` or `../mychan`). Can be used multiple times.
</ParamField>

<ParamField path="--use-local" type="boolean">
  Use locally built packages. Identical to `-c local`.
</ParamField>

<ParamField path="-O, --override-channels" type="boolean">
  Do not search default or .condarc channels. Requires --channel.
</ParamField>

<ParamField path="--repodata-fn" type="string">
  Specify file name of repodata on the remote server. Can be passed multiple times.
</ParamField>

<ParamField path="--experimental" type="string">
  Enable experimental features. Options: `jlap` (download incremental package index data), `lock` (use locking when reading/updating index cache).
</ParamField>

<ParamField path="--no-lock" type="boolean">
  Disable locking when reading, updating index (repodata.json) cache.
</ParamField>

<ParamField path="--repodata-use-zst / --no-repodata-use-zst" type="boolean">
  Check for/do not check for repodata.json.zst. Enabled by default.
</ParamField>

## Package Linking and Install-time Options

<ParamField path="--clobber" type="boolean">
  Allow clobbering of overlapping file paths within packages, and suppress related warnings.
</ParamField>

<ParamField path="--copy" type="boolean">
  Install all packages using copies instead of hard- or soft-linking.
</ParamField>

<ParamField path="--no-shortcuts" type="boolean">
  Don't install start menu shortcuts.
</ParamField>

<ParamField path="--shortcuts-only" type="string">
  Install shortcuts only for this package name. Can be used several times.
</ParamField>

## Networking Options

<ParamField path="-C, --use-index-cache" type="boolean">
  Use cache of channel index files, even if it has expired. This is useful if you don't want conda to check whether a new version of the repodata file exists.
</ParamField>

<ParamField path="-k, --insecure" type="boolean">
  Allow conda to perform "insecure" SSL connections and transfers. Equivalent to setting `ssl_verify` to `false`.
</ParamField>

<ParamField path="--offline" type="boolean">
  Offline mode. Don't connect to the Internet.
</ParamField>

## Output, Prompt, and Flow Control Options

<ParamField path="--download-only" type="boolean">
  Solve an environment and ensure package caches are populated, but exit prior to unlinking and linking packages into the prefix.
</ParamField>

<ParamField path="--show-channel-urls" type="boolean">
  Show channel urls. Overrides the value given by `conda config --show show_channel_urls`.
</ParamField>

<ParamField path="--json" type="boolean">
  Report all output as json. Suitable for using conda programmatically.
</ParamField>

<ParamField path="-v, --verbose" type="boolean">
  Can be used multiple times. Once for detailed output, twice for INFO logging, thrice for DEBUG logging, four times for TRACE logging.
</ParamField>

<ParamField path="-q, --quiet" type="boolean">
  Do not display progress bar.
</ParamField>

<ParamField path="-d, --dry-run" type="boolean">
  Only display what would have been done.
</ParamField>

<ParamField path="-y, --yes" type="boolean">
  Sets any confirmation values to 'yes' automatically. Users will not be asked to confirm any adding, deleting, backups, etc.
</ParamField>

## Environment Protection

<ParamField path="--override-frozen" type="boolean">
  DANGEROUS. Use at your own risk. Ignore protections if the environment is frozen.
</ParamField>

## Examples

<CodeGroup>
```bash Update single package
conda update numpy
```

```bash Update in specific environment
conda update -n myenv scipy
```

```bash Update all packages
conda update --all
```

```bash Update conda itself
conda update conda
```

```bash Update multiple packages
conda update numpy pandas matplotlib
```

```bash Update without changing dependencies
conda update --no-update-deps scikit-learn
```

```bash Update from specific channel
conda update -c conda-forge tensorflow
```

```bash Dry run to preview updates
conda update --dry-run --all
```
</CodeGroup>

## Common Use Cases

### Updating All Packages in an Environment

Update all packages to their latest compatible versions:

```bash
conda update --all
```

Or for a specific environment:

```bash
conda update -n myenv --all
```

### Updating Conda Itself

Keep your conda installation up to date:

```bash
conda update conda
```

### Updating Specific Packages

Update one or more specific packages:

```bash
conda update numpy pandas matplotlib
```

### Updating Without Breaking Dependencies

Use `--no-update-deps` to update only the specified package:

```bash
conda update --no-update-deps requests
```

### Checking What Would Be Updated

Use dry-run mode to preview changes:

```bash
conda update --all --dry-run
```

### Updating with Dependency Updates

Explicitly update dependencies:

```bash
conda update --update-deps scipy
```

<Note>
  If you wish to prevent existing packages from updating, use the `--no-update-deps` option. This may force conda to install older versions of the requested packages, and it does not prevent additional dependency packages from being installed.
</Note>

<Warning>
  Using `--no-deps` may result in an environment with incompatible packages. Use this option with great caution.
</Warning>
